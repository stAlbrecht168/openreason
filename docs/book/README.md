# The OpenReason Book

This is the educational documentation for OpenReason — written for people who want to understand the ideas behind the system, not just run it.

You do not need to be a software developer to read this book. The concepts here belong to analysis, argumentation theory, linguistics, and epistemology. The code is an implementation of those concepts, not the point.

## How to read this book

**If you are new to OpenReason**, start with chapter 1 and read in order. By the end of chapter 3, you will understand the core ideas. Chapters 4 through 7 go deeper.

**If you understand the evidence model already**, skip to chapter 3 (packs) or chapter 7 (the full worked example).

**If you want to contribute a framework**, read chapters 3, 4, and 6 in that order.

**If you want to understand how Claude Code fits in**, chapter 5 is the place to start.

---

## What this book is not

This book does not teach you which analytical conclusions to reach. It teaches you how to make your reasoning visible — and how to read analytical claims critically, whoever makes them.

OpenReason does not determine what is true. It provides a method for showing what is observed, what is inferred, what is interpreted, and what remains uncertain.
